# 문제 정의

## 비정형 데이터의 학습 방법

- 텍스트: 형태소 단위 분리/학습
- 이미지: 일정한 사이즈 필터로 학습

## 파악되지 않은 연구/해결이 필요한 경우

규칙을 찾아내는 AI 능력을 활용한다.

- 지도학습: 정답이 있는 과거 데이터를 기반으로 예측
- 비지도학습: 데이터 사이의 관계를 파악하고 특성을 발견
- 강화학습: 행동에 대한 보상으로 예상하지 못한 해법 발견

## AI의 주요 기능

- 예측(Prediction) : 데이터 학습, 패턴 파악을 통한 결과 예측
- 생성(Generation) : 텍스트, 음성, 사진, 영상 등을 생성
- 소통(Conversation) : 대화기능 기반으로 답변과 질문 연출
- 자동화(Automation) : 더 효율적인 방법을 찾아 최적화
- 인식(Recognition) : 대상의 다양한 특성을 인지하고 상황판단

해당 5가지 기능으로 정의하고, 이런 기능으로 정의할 수 없다면 AI로 해결하기 적합한 문제인지 다시 고민해볼 필요가 있다.

# 데이터 수집

## 데이터 수집 계획

데이터 수집 시 반복 작업 및 수집 지연 방지를 위한 정의, 방안수립

- 필요한 데이터 정의
- 구체적인 수집 방안 수립

## 데이터 수집 종류

수집 위치 기준

| 유형        | 설명                                                                                                 | 종류                                                             |
| ----------- | ---------------------------------------------------------------------------------------------------- | ---------------------------------------------------------------- |
| 내부 데이터 | - 내부 시스템/업무 영역에 위치<br>- 데이터 수명 주기 관리가 용이<br>- 민감 정보가 포함될 경우가 있음 | - 서비스(인증/거래)<br>- 네트워크(시스템)<br>- 마케팅(VOC, 판매) |
| 외부 데이터 | - 외부시스템/업무 영역에 위치<br>- 데이터 구매 혹은 수집절차 필요<br>- 공개된 데이터                 | - 소셜(SNS)<br>- 공공(의료, 기상)                                |

## 데이터 수집

- 내부 데이터: 내 영역의 데이터로 보유 중이므로, 담당자 간 협의를 통한 확보 및 수집 가능
- 외부 데이터: 별도의 수집 방법 활용 필요 (크롤링, RSS, Open API, Streaming)

대표적인 데이터 수집 방식

1. 크롤링: 다양한 웹문서 및 콘텐츠를 수집하는 방식
2. Open API: 공개 API를 이용해서 데이터 수집, 공공데이터 포털 등 활용
3. RSS: 웹사이트에 게시된 새로운 글을 공유하는 프로토콜
4. 스트리밍: 미디어 데이터의 실시간 수집

수집하기 전에 검토 사항

- 분석 목적에 부합되는 데이터인지
- 대표성을 가질 수 있는 데이터인지
- 충분히 많고 **깨끗한 데이터**인지

여기서 깨끗한 데이터란?

- 결측치(손실되거나 빈 값)가 없고, 편향되지 않은 데이터를 의미함.
- 편향된 경우, 해당 편향 비율만큼 정확도가 올라가서 편향된 결과 도출
- 결측치는 판단 불가, 결측치가 많을 경우, 편향된 결과 도출.
