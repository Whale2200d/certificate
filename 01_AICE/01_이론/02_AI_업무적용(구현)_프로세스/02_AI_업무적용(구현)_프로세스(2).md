# 데이터 분석

데이터 기초 탐색(분포/타입 등)하여 데이터의 가공 방향을 결정한다.

## 데이터 타입

- 수치형, 문자형, 범주형, 불린형
  - 수치형: 연속형과 이산형으로 구분 (예시: 키, 몸무게 / 사람 수 등)
  - 문자형: 문자, 문자+숫자형으로 구분
  - 범주형: 범주를 나눌 수 있음. 사칙연상 불가능. 순서형과 명목형 (예시: 학점, 만족도 / 성별, 업체명)
  - 불린형: 참/거짓

- 수치형으로 변경 가능한 범주형
  - 등급(Grade)과 계약 물량이 유사성을 가지므로, 수치형으로 변경 가능
  - 업체명은 순서가 없기 때문에 수치형으로 바꾸는 것이 더 나쁠 수 있음. 이 경우, 인코딩을 사용한다.
  - 인코딩은 전처리 파트에서 추가로 학습 예정이다.

## 기술(descriptive, 묘사/설명) 통계

통계적인 방법으로 데이터를 요약하고, 데이터를 설명하고자 할 때 사용한다. (통계적 방법 -> 데이터 요약 -> 데이터 설명)

- 데이터가 얼마나 모여있는지, 얼마나 흩어져 있는지

1. 얼마나 데이터가 모여 있는가?
   - 개수, 평균값, 중앙값, 최빈값
   - 개수: 데이터의 개수
   - 평균값: 각 데이터를 모두 더한 후 데이터 개수로 나눈 값
   - 중앙값(중위값, 중간값): 데이터를 크기 순으로 배열 시, 가운데 위치한 값
   - 최빈값: 데이터 중 가장 빈도가 높은 값
2. 얼마나 데이터가 흩어져 있는가?
   - 최대값/최소값, 분산/표준편차, 사분위수, 첨도/왜도
   - 최대값/최소값: 데이터 중 가장 큰/작은 값
   - 분산/표준편차: 데이터가 평균적으로 떨어진 정도 (평균 간 차이의 제곱의 평균 / 분산의 제곱근)
   - 사분위수: 모든 데이터를 크기 순으로 배열 시 4등분한 지점에 있는 값
   - 첨도/왜도: 정규분포 대비 뾰족한/비대칭한 정도 (정규분포 대비 뾰족할수록 첨도 > 0, 왼쪽으로 모일수록 왜도 > 0, 오른쪽으로 모일수록 왜도 < 0)

## 데이터 시각화

- 숫자 문자로 되어있는 분석보다 **시각화된 결과**를 통해 더욱 쉽게 이해할 수 있다.
- 시각화를 통해 데이터 간 상관관계를 깊게 분석하는 탐색적 데이터 분석, EDA(Explorative Data Analysis) 수행한다.
- 데이터 시각화를 통해 얻을 수 있는 결과는 2가지가 있다.
  - Feature 간 상관관계 분석 (Scatter plot(산점도), Heat map)
  - Feature 자체 분석 (Box plot(박스 차트), Dist plot(분포 차트), Count plot)

- 산점도: 두 컬럼의 수치값을 좌표평면상의 숫자로 표현
  - 산점도 분석을 통해 Feature 간의 음/양의 상관관계를 확인할 수 있다.
  - 다만 시각적으로 상관관계 파악이 쉽지 않다.

- Heat map: -1~1의 값을 가지는 상관계수를 색상으로 표현한 지도
  - 산점도의 한계로 Heat map을 많이 사용한다.
  - Heatmap 분석 시, Label Data와 연관관계가 **높은 칼럼들은 유지(학습 데이터로 사용)**, **연관도가 낮은 칼럼들은 Drop**하는 방식으로 Column의 차원수를 줄여나가는 등 의사결정 필요

- Box plot: 데이터 통계분포를 박스모양으로 나타내는 기법
  - Column별 이상치 판단할 때 주로 사용한다.
  - 예시: IQR smoothing
    - IQR = Q3 - Q1
    - Upper Fence = Q3 + (IQR \* 1.5)
    - Lower Fence = Q1 - (IQR \* 1.5)
    - Upper Fence, Lower Fence 바깥 값을 이상치로 정의

- Dist plot: 수치형 데이터에 대해 구간별 빈도수를 나타내는 기법
  - Hue(색상)을 class별 구분하여 각 클래스에 대해 해당 데이터의 빈도 파악 가능

- Count plot: 범주형 데이터의 class별 데이터 빈도수
  - 예시: Business, Eco, Eco Plus
